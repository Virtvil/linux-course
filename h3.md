## Materiaalit

### Apache installed with Ansible

(_Huom! Käyn tämän osion tarkemmin läpi soveltaen sen toimimaan nginx:ään myöhemmässä "Ansible Nginx"-osiossa_)

- Apache2 ja nginx-tyyppiset verkkopalvelut voidaan asentaa myös Ansiblen avulla.
- Määritystä luodessa tärkeässä asemassa ovat kolme kansiota:
  - _Files_, sisältää master-kopiot tiedostoista. (Tähän on hyvä lisätä kommentti jolla kerrotaan käyttäjille, että kaikki kyseiseen conf-tiedostoon tehdyt muutokset tullaan päällekirjoittamaan)
  -  _Handlers_, käynnistää palvelut uudelleen muokkausten jälkeen.
  -  _Tasks_, sisältää suurimman osan koodista.
- On hyvä varmistaa muutoksien käyttöönotto demonin uudelleenkäynnistämisellä _config_-tiedostojen muuttamisen jälkeen.

### Handlers - Ansible-doc

- Joskus palvelinten kanssa työskennellessä halutaan toimia niin, että jokin tietty tehtävä suoritetaan vain, kun koneeseen tehdään muutos. 
- Käyttäjä saattaa haluta käynnistää jonkin tietyn palvelun uudelleen, jos suoritettävä tehtävä päivittää kyseisen palvelun, mutta ei päivitä tätä jos kokoonpano on muuttumaton.
- Suoritettavat tehtävät voivat ohjeistaa yhden tai useamman _handlerin_ suorittamaan ilmoitusavainsanalla, eli _notify_ -merkinnällä. Tätä _notify_-avainsanaa voidaan käyttää suoritettavassa tehtävässä, ja se hyväksyy luettelon nimistä, joille ilmoitetaan tehtävän suorittamista muutoksista.
- _Handlers_ merkitään suoritettavaan tiedostoon omana osionaan, ja tämän alle nimetyt tehtävät suoritetaan siinä järjestyksessä missä ne ovat listattu. (_Handlers_ määrittää siis suoritusjärjestyksen, ei _notify_)

### Ansible-doc service

- Ohjaa palveluita isäntälaitteella.

**enabled**
- Määrittää käynnistyykö määritelty ohjelma laitteen käynnistyesssä.

**name**
- Ohjelman/palvelun nimi.

**state**
- Ohjelman/palvelun tila.
- Määrittää ajettaessa halutaanko ohjelman olevan päällä vai ei.

**Esimerkit:**

- name: Start service httpd, if not started

  ansible.builtin.service:

    name: httpd

    state: started

- name: Stop service httpd, if started

   ansible.builtin.service:

    name: httpd

    state: stopped

- name: Restart service httpd, in all cases

  ansible.builtin.service:

    name: httpd

    state: restarted

- name: Reload service httpd, in all cases

  ansible.builtin.service:

    name: httpd

    state: reloaded

- name: Enable service httpd, and not touch the state

  ansible.builtin.service:

    name: httpd

    enabled: yes

- name: Start service foo, based on running process /usr/bin/foo

  ansible.builtin.service:

    name: foo

    pattern: /usr/bin/foo

    state: started

- name: Restart network service for interface eth0

  ansible.builtin.service:

    name: network

    state: restarted

    args: eth0

## Apache2

Aloitin tehtävät poistamalla molemmat Apache2 ja nginx-asennukset jotka olivat koneellani, käyttämällä komentoa _sudo apt-get purge Apache2_ ja _sudo apt-get purge nginx_. Purge komento poisti myös kaikki konfiguraatio-tiedostoni, jotka _remove_-komento olisi jättänyt.

Asennettuani Apache2 komennoilla _sudo apt update_ ja _sudo apt install apache2_ ryhdyin asettamaan palomuurejani sallimaan Apache2 liikennettä. Avasin portin 80 käyttämällä komentoa _sudo ufw allow 'WWW'_ ja tarkistin tämän olevan auki komennolla _sudo ufw status_. Todettuani tämän olevan päällä tarkistin Apache2 asennuksen komennolla _sudo systemctl status apache2_, jonka vielä tarkistin siirtymällä verkkoselaimelleni ja avaamalla http://localhost sivuston.

Apache2 Landing Page

Ryhdyin seuraavaksi luomaan omaa verkkosivuani (_vantaa.fishfin.com_) tehden tälle kansion komennolla _sudo mkdir -p /var/www/vantaa.fishfin.com_, asetin kansiolle oikeudet komennolla _sudo chmod -R 755 /var/www/vantaa.fishfin.com_ ja loin simppelin html-tiedoston komennolla _nano /var/www/vantaa.fishfin.com/index.html_

**html-tiedosto:**

<html>
  
    <head>
    
        <title>Hello World!</title>
        
    </head>
    
    <body>
    
        <h1>Toimiikohan tää vielä?</h1>
        
    </body>

</html>

Seuraavaksi loin config-tiedoston sivulle käyttäen komentoa _sudo nano /etc/apache2/sites-available/vantaa.fishfin.com.conf_


**Config-tiedosto:**

<VirtualHost *:80>
 
    ServerName vantaa.fishfin.com
    
    DocumentRoot /var/www/vantaa.fishfin.com

    <Directory /var/www/vantaa.fishfin.com>
    
        Require all granted
    
    </Directory>

</VirtualHost>

Aktivoin sivuni komennolla _sudo a2ensite vantaa.fishfin.com.conf_ ja poistamalla Apache2 Default Landing Page käytöstä komennolla _sudo a2dissite 000-default.conf_.

Unohdin aluksi antaa config-tiedostolle _<Directory>_-osion luvat, joka aiheutti hieman ongelmia. Tämän lisäksi muutin myös _hosts_-tiedostoani, lisäämällä tänne uuden verkkosivun (_sudo micro /etc/hosts_):

127.0.0.1   vantaa.fishfin.com

Tämän jälkeen käynnistin Apache2 uudelleen komennolla _sudo systemctl reload apache2_ testasin luomani tiedostot komennolla _sudo apache2ctl configtest_. Seuraavaksi tarkistinkin toimivuuden siirtymällä verkkosivulleni ja osoitteeseen http://vantaa.fishfin.com.

## Nginx

Aloitin Nginx asennuksen ottamalla Apache2-demonin pois päältä komennolla _sudo service apache2 stop_ ja asentamalla Nginx kaikille tutuilla _sudo apt update_ & _sudo apt install nginx_ komennoilla.
Aluksi loin uuden _mypage.html_-tiedoston _/var/www/html/_-hakemistoon.


**mypage.html-tiedosto:**


<!DOCTYPE html>

<html>
  
    <head>
    
        <title>Heippa maailma!</title>
        
    </head>
    
    <body>
    
        <h1>Nginx revving up</h1>
        
    </body>

</html>

Seuraavaksi suuntasin _/etc/nginx/sites-available/_-hakemistoon, jossa avasin _default_-tiedoston, johon lisäsin _location / {_-kohdan alle uuden rivin: _index mypage.html;_. Suoritin _sudo nginxx -t_-komennon joka varmisti konfiguraatio-tiedoston syntaxin olevan kunnossa ja suoritin Nginx-demonin uudelleen _sudo nginx -s reload_-komennolla. Tämän jälkeen kurkistin localhostiani, joka näytti edelleen Apache2 aloitussivua.... Yritin uudelleen selaimeni incognito-ikkunalla ja kappas! Sivuni toimi, eikä aikaakaan mennyt kuin ehkä 1/10-osa verrattuna Apache2...
