### Use of DSL Functions in Case Configuration
- Tutkiessa Mozilla Engineering manifesteja, toimintoja ja ohjausrakenteita, selvisi kuinka paljon turhia funktioita tämä käyttää.
- Analysoidessa United States Government Configuration Baseline (USGCB) selvisi tiedostojen käsittelyn Augeasilla olleen palvelun yleisin toiminto.
- Analyysi osoittaa, että vaikka johtavat CM:t tarjoavat huomattavan määrän toimintoja järjestelmän manipuloimiseen, vain pieni määrä toiminnoista näyttää kattavan suurimman osan käyttötapauksista.

### Dependencies Between Main Functions
- Yleisimpiä toimintoja ovat pakettitiedostopalvelu, käyttäjä, ryhmä ja suorittaja.
- Yleisiä käyttötapauksia ovat demoni- ja sovellusmääritykset.
- Joissakin tapauksissa käyttäjät ja ryhmät on luotava teknisistä syistä, vaikka ihmiskäyttäjiä hallittaisiinkin CM:n ulkopuolella.
- Järjestelmän tulee olla idempotentti. Tämä tarkoittaa, että muutoksia järjestelmään tehdään vain jos järjestelmä ei ole jo halutussa tilassa.

## Räpylä:
Päätin tässä tehtävässä asentaa itselleni _Caddy_-ohjelman, sillä tunnilla olin asentanut itselleni _Postgres_-ohjelman. Aloitin tehtävän antamalla tutut komennot:

```
sudo systemctl disable nginx
sudo systemctl stop nginx
```
Joilla pysäytin käynnissä olevan _nginx_-ohjelman. Tämän jälkeen asensin itselleni _Caddyn_ antamalla komennot:
```
sudo apt update
sudo apt install caddy
```
Seuraavaksi käynnistin ohjelman simppelillä _caddy run_ komennolla, jonka jälkeen avasin itselleni toisen terminaalin. Loin terminaalissani itselleni caddy-nimisen kansion, johon loin _caddy.json_ tiedoston, josta tein seuraavanlaisen:
```
{
	"apps": {
		"http": {
			"servers": {
				"example": {
					"listen": [":2015"],
					"routes": [
						{
							"handle": [{
								"handler": "static_response",
								"body": "Hello, world!"
							}]
						}
					]
				}
			}
		}
	}
}

```

Siirsin halutun tiedoston Caddyn konfiguraatioon komennolla: 
```
curl localhost:2019/load \
	-H "Content-Type: application/json" \
	-d @caddy.json
```

Ja tarkistin sen perille menon komennolla _curl localhost:2019/config/_ ja varmistin toimivuuden komennolla _curl localhost:2015_, joka palautti minulle _"Hello, world!"_-tekstin.
