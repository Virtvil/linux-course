## Materiaalit:
- Pelkän versionhallinnan sijaan Git on pikemminkin minitiedostojärjestelmä, jonka päälle on rakennettu muita tehokkaita työkaluja.
- Joka kerta tiedostoja hyväksyessä ja lisätessä (commit), tai projektin tilaa päivittäessä, Git periaatteessa ottaa kuvan siitä miltä tiedostosi näyttävät sillä hetkellä ja tallentaa viittauksen niinkutsuttuun tilannekuvaan (snapshot).
- Git tallentaa tiedostoista useita eri tilannekuvia, mutta vain jos tiedostoja muutetaan. Tiedostojen olleena muuttumattomina näitä ajettaessa Gitin kautta, viittaa Git vain muistissaan olevaan aiempaan samaan versioon tiedostosta.
- Gitin tarvitsee toimintojaan varten usein vain paikallisia tiedostoja ja resursseja toimiakseen. Tämän vuoksi projekteja tarkastellessa projektin historian näkee lähes välittömästi.
- Paikallisten tiedostojen myötä on näihin helppo tehdä muutoksia, jotka pystytään lataamaan verkkoon tämän ollessa saatavilla. Työskentely on siis helppoa myös ilman verkkoyhteyttä, tiedostojen tallentuessa paikallisesti.
- Kaikki Gitissä oleva tarkistetaan eräänlaiseksi tarkistussummaksi ennen kuin se tallennetaan, jonka jälkeen siihen viitataan kyseisellä tarkistussummalla. Tämä tarkoittaa, ettei tiedostoja tai hakemistojen sisältöä ole mahdollista muuttaa Gitin tietämättä siitä. Tämä toiminto on sisäänrakennettu Gitiin alimmilla tasoilla ja on olennainen osa sen filosofiaa. Tietojen menettäminen siirron aikana tai tiedostojen vioittuminen ilman että Git havaitsee sen on täten mahdotonta.
- Kaikki toiminnot Gitissä lähinnä vain lisäävät dataa Gitin tietokantaan.
- Gitillä on kolme päätilaa, joissa tiedostot ovat: _modified, staged,_ sekä _committed_:
  - _Modified_ tarkoittaa, että tiedostoa on muokattu, mutta muutosta ei ole vielä sidottu tietokantaan.
  - _Staged_ tarkoittaa, että muokattu tiedosto on merkitty sen nykyiseksi versioksi ja on valmis siirrettäväksi seuraavaan tilannekuvan päivitykseen.
  - _Committed_ tarkoittaa, että tiedot on tallennettu turvallisesti paikalliseen tietokantaan.
